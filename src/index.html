What this validator checks:

The validator check the rules one by one and don't verify next one if the previous one isn't correct.

  • XSD Schema comparison with given XML
  • Check the elements' IDs in the XML are unique:
    • bus-interface → network-id
    • telegram → com-id
    • data-set → id
    • com-parameter → id
    • source & destination → id
  • Verify that the links between the telegram IDs are correct (other referenced ID exist):
    • com-id → data-set id
    • com-id → com-parameter id
